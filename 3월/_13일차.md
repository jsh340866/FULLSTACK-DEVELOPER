  DAY 07 CSS 참조
  
  1. MQ의 media방식

     - media 방식
     -   /* 태블릿 width : 1024  */
    @media all and (max-width:1024px){
      div{background-color: green; height: 100px;}}
     - CSS 파일 하나(또는 <style> 태그 안)에 모든 기기의 설정을 다 적는 방식이에요.
     - @media 방식은 "한 집(파일) 안에서 방을 나누는 것"

  2. MQ의 link 방식    
     - link 방식
     -      <!-- DESKTOP -->
     - \<link rel="stylesheet" href="./css/desktop.css">
     - 기기별로 CSS 파일을 완전히 분리해서 관리. (desktop.css, tablet.css 등)
     - <link> 방식은 "용도에 따라 별개의 집(파일)을 짓는 것"

  3. JS 코드
     - console.log() : 개발자 브라우저 로그창에 출력하기 위한 기본적인 형태
     - alert(): 브라우저 상단에 작은 알림창을 띄워주는 함수
     - document.write() : 화면에 직접 쓰기
     - innerHTML : 특정 부분만 교체하기
     - typeof : 데이터 정체 확인하기
     - null vs undefined : 비어있음의 차이
         - null: 개발자가 "여기는 의도적으로 비워둘게!"라고 직접 값을 넣은 상태예요.
         - undefined: 변수만 만들어놓고 값을 안 넣어서, 자바스크립트가 "뭔지 모르겠어"라고 내뱉는 상태예요.
     - 쓰는 법 (write), 바꾸는 법 (innerHTML), 검사하는 법 (typeof), 비우는 법 (null/undefined)
