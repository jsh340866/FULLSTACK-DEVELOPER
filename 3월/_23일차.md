    *** 04_지오코더(장소표시) ***
    - 지오코더는 우리가 아는 '주소'와 컴퓨터가 아는 '좌표' 사이를 번역해 주는 번역기라고 생각하면 돼요!
    - https://nominatim.org/ 참고 사이트
    
    // 지도 우클릭 이벤트
    map.on('contextmenu', function (e) {
        // console.log(e.latlng);
        const lat = e.latlng.lat;
        const lng = e.latlng.lng;
        console.log(lat, lng);

        // 비동기 요청 처리함수 (fetch)
        fetch(`https://nominatim.openstreetmap.org/reverse?format=jsonv2&lat=${lat}&lon=${lng}&layer=address`)
            .then((response) => { return response.json(); })	//성공시
            .then((data) => {
                // console.log(data);
                const displayName = data && data.display_name
                const suberb = data && data.address.suberb;
                const village = data && data.address.village;
                console.log(displayName, suberb, village);

                L
                    .popup()
                    .setLatLng(e.latlng)
                    .setContent(`<div style='width:220px;'>
                    위도 : ${lat}<br/>
                    경도 : ${lng}<br/>
                    주소 : 
                    <a 
                        href='https://search.naver.com/search.naver?where=nexearch&sm=top_hty&fbm=0&ie=utf8&query=${displayName}'
                        target='_blank'
                    >
                        ${displayName}
                    </a>
                    <br/>
                    
                    </div>`)
                    .openOn(map);


            })	//성공시
            .catch((error) => { console.log(error) }) 		//실패시
    });

    *** 05_클러스터링 ***
    - 지도 위에 너무 많은 마커가 있을 때, 이를 그룹화하여 숫자로 표시해 주는 기술
    
    	//---------------------------------------
		// [1] 랜덤 마커 생성 (클러스터링 없이 리스트 저장)
		//---------------------------------------

		const markerList = []; // 생성된 마커들을 저장할 리스트
		const centerLat = 35.872638;
		const centerLng = 128.600690;

		function getRandomOffsetIrregular(maxOffset) {
			const base = (Math.random() * Math.random()) * maxOffset;
			const sign = Math.random() < 0.5 ? -1 : 1;
			return base * sign;
		}

		for (let i = 0; i < 500; i++) {
			const latOffset = getRandomOffsetIrregular(0.06);
			const lngOffset = getRandomOffsetIrregular(0.06);
			const lat = centerLat + latOffset;
			const lng = centerLng + lngOffset;

			const marker = L.marker([lat, lng]).bindPopup(`
				<div>
					<strong>랜덤 마커 #${i + 1}</strong><br>
					위도: ${lat.toFixed(6)}<br>
					경도: ${lng.toFixed(6)}
				</div>
			`);
			markerList.push(marker); // 리스트에 저장만 함
		}
		console.log('markerList', markerList);

		//---------------------------------------
		// [2] 클러스터링 적용 (저장된 마커 사용)
		//---------------------------------------
		/*
			maxClusterRadius는 픽셀 단위로,
			지도의 줌 레벨에 따라 실제 거리(m)는 달라집니다.
			예: 줌레벨 12에서 약 500~800m
		*/
		const markerClusterGroup = L.markerClusterGroup({
			maxClusterRadius: 100  // ← 클러스터링 반경 (픽셀 기준)
		});

		// 저장된 마커를 클러스터 그룹에 추가
		markerList.forEach(marker => {
			markerClusterGroup.addLayer(marker);
		});

		// 클러스터 그룹을 지도에 추가
		map.addLayer(markerClusterGroup);

    *** BS5 Layout ***
	DAY_17_BS5 참조

    1. Container (컨테이너): 전체를 감싸는 가장 바깥쪽 틀이에요. 콘텐츠의 양옆 여백을 조절해 주죠.
        - .container: 좌우 여백이 있는 고정 폭
        - .container-fluid: 화면 꽉 차는 100% 폭
        
    2. 반응형 중단점 (Breakpoints)
        - 부트스트랩의 진짜 위력은 화면 크기에 따라 배치가 자동으로 바뀌는 것에 있어요!
        
    클래스명        화면 크기 (너비)     대상 기기
    .col-           < 576px            모바일 (세로)
    .col-sm-        ≥ 576px            모바일 (가로)
    .col-md-        ≥ 768px            태블릿
    .col-lg-        ≥ 992px            노트북 / 작은 모니터
    .col-xl-        ≥ 1200px           데스크탑

    3. 사용예시
        <main class="container mt-5">
      <div class="row">
        <section class="col-12 col-md-4 border">첫 번째 섹션</section>
        <section class="col-12 col-md-4 border">두 번째 섹션</section>
        <section class="col-12 col-md-4 border">세 번째 섹션</section>
      </div>
    </main>
