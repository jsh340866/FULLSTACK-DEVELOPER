# 26-04-20

## 프로그래밍 언어 - Java 형변환(Type Conversion) 심화

## 오늘자 주요 태그

* **자동 형변환**: Promotion / 작은 타입 → 큰 타입 / 데이터 손실 없음
* **강제 형변환**: Casting / 큰 타입 → 작은 타입 / 데이터 손실 주의
* **연산식 형변환**: `int` 미만 자료형 연산 결과는 `int` / 문자열 결합 연산

## 오늘 배운 것

* **중요**: `C01TypeChange.java` ~ `C07TypeChange_String.java`
* **알아두기**: `byte`, `short`, `char` 연산 시 자바는 내부적으로 `int`로 변환하여 처리함
* **형변환 시 주의점**: 정수 간 나눗셈 결과는 정수이며, 실수를 얻으려면 피연산자 중 하나를 실수로 형변환해야 함

### Java 형변환 핵심 정리

#### 1. 자동 형변환 (Automatic Type Conversion)
* **개념**: 컴파일러에 의해 자동으로 일어나는 형변환으로, 큰 자료형 공간에 작은 자료형 값을 넣을 때 발생해요.
* **크기 순서**: `byte` < `short`, `char` < `int` < `long` < `float` < `double`
* **특징**: 데이터 손실이 최소화되는 방향으로 자료형이 결정돼요.
    * **예시**: `int intValue = byteValue;` (자동 변환 가능)
    * **주의**: `char` 타입은 음수를 포함하지 않기 때문에 `byte`에서 `char`로의 자동 변환은 불가능해요.

#### 2. 강제 형변환 (Explicit Casting)
* **개념**: 큰 자료형을 작은 자료형 공간에 저장할 때 사용하며, 캐스팅 연산자 `()`를 명시해야 해요.
* **주의사항**: 데이터 손실(비트 잘림)이 발생할 수 있으니 주의가 필요해요.
    * **예시**: `int`를 `byte`로 변환 시 앞부분의 비트가 제거되어 값이 변할 수 있음
    * **실수 → 정수**: 소수점 이하 자리가 버려짐 (예: `3.14` → `3`)

#### 3. 연산식에서의 타입 변환
* **정수 연산**: `byte`, `short`, `char` 타입의 변수는 연산 시 자동으로 `int`로 변환되어 계산돼요.
    * `short result = s1 + s2;`와 같은 코드는 컴파일 에러가 발생하며, 결과값이 `int`로 나오기 때문이에요.
* **실수 연산**: 피연산자 중 하나가 `double`이면 다른 피연산자도 `double`로 변환되어 연산 결과가 나와요.

### 문자열과 타입 변환 (`C07TypeChange_String.java`)

#### 1. 문자열 결합 연산 (+)
* **개념**: 문자열과 `+` 연산을 하면 다른 피연산자도 문자열로 변환되어 결합이 일어나요.
* **연산 순서의 중요성**:
    * `2 + 3 + ""` → `"5"` (숫자 합 계산 후 문자열 결합)
    * `"" + 2 + 3` → `"23"` (처음부터 문자열로 결합 시작)

#### 2. 문자열을 기본 타입으로 변환
* 웹 개발 시 사용자가 입력하는 값은 기본적으로 문자열이므로, 이를 숫자로 변환하는 과정이 매우 중요해요.
* **주요 메소드**:
    * `Integer.parseInt("100")`
    * `Double.parseDouble("3.14")`
    * `Byte.parseByte("10")`
* **문자 추출**: 문자열에서 특정 인덱스의 문자를 가져오고 싶을 때는 `charAt(0)`을 활용해요.
