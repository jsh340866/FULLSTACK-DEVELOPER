# 1. 메서드 오버로딩 (Overloading)

## 문제 상황

덧셈 기능이 필요한데 숫자가 2개일 수도 있고 3개일 수도 있습니다.  
메서드 이름을 전부 다르게 만들어야 할까요?

## 해결 방법

같은 이름을 사용하고 매개변수만 다르게 만들면 됩니다.

```java
class Calc {

    int sum(int a, int b) {
        return a + b;
    }

    int sum(int a, int b, int c) {
        return a + b + c;
    }

    double sum(double a, double b) {
        return a + b;
    }
}
```

## 사용 예시

```java
Calc c = new Calc();

System.out.println(c.sum(10, 20));       // 30
System.out.println(c.sum(10, 20, 30));   // 60
System.out.println(c.sum(1.5, 2.5));     // 4.0
```

## 핵심 정리

* 이름은 같아도 매개변수가 다르면 다른 메서드입니다.
* 기능은 같고 입력 형태만 다를 때 사용합니다.

# 2. 가변 인자 (Variable Arguments)

## 문제 상황

숫자를 몇 개 받을지 매번 다릅니다.

## 해결 방법

```java
int ...args
```

형태로 받습니다.

## 예시

```java
class Calc {

    int sum(int ...args) {

        int total = 0;

        for(int n : args) {
            total += n;
        }

        return total;
    }
}
```

## 사용 예시

```java
Calc c = new Calc();

System.out.println(c.sum(1,2));         // 3
System.out.println(c.sum(1,2,3,4));     // 10
System.out.println(c.sum(5,5,5,5,5));   // 25
```

## 핵심 정리

* 내부적으로 배열처럼 처리됩니다.
* 개수가 일정하지 않은 입력에 유용합니다.

# 3. 생성자 (Constructor)

## 문제 상황

학생 객체를 만들 때 이름과 나이를 바로 넣고 싶습니다.

## 해결 방법

생성자를 사용합니다.

```java
class Student {

    String name;
    int age;

    Student(String n, int a) {
        this.name = n;
        this.age = a;
    }
}
```

## 사용 예시

```java
Student s1 = new Student("홍길동", 20);
Student s2 = new Student("김철수", 22);
```

## 핵심 정리

* 객체 생성 시 자동 실행됩니다.
* 초기값 세팅에 사용됩니다.

# 4. this 와 this()

## 문제 상황

매개변수 이름과 멤버변수 이름이 같으면 구분이 어렵습니다.

## 해결 방법

```java
this.name
```

현재 객체의 변수라는 뜻입니다.

## 예시

```java
class Student {

    String name;

    Student(String name) {
        this.name = name;
    }
}
```

---

## 생성자끼리 연결하기

```java
class Student {

    String name;
    int age;

    Student() {
        this("무명", 0);
    }

    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

## 핵심 정리

* `this.변수명` → 내 객체 변수
* `this()` → 다른 생성자 호출

# 5. 접근 제어자

## 문제 상황

누구나 마음대로 값을 바꾸면 위험합니다.

예: 나이를 -100으로 변경

## 해결 방법

```java
private
```

으로 숨깁니다.

## 예시

```java
class Person {

    private int age;
}
```

## 종류

| 접근제어자 | 접근 범위 |
|---|---|
| public | 어디서나 가능 |
| private | 자기 클래스 내부만 가능 |
| protected | 상속 관계 포함 |
| default | 같은 패키지 |

# 6. Getter / Setter

## 문제 상황

`private` 으로 막아두면 값을 못 읽고 못 바꿉니다.

## 해결 방법

메서드로 통제합니다.

```java
class Person {

    private int age;

    public void setAge(int age) {
        if(age >= 0) {
            this.age = age;
        }
    }

    public int getAge() {
        return age;
    }
}
```

## 사용 예시

```java
Person p = new Person();

p.setAge(25);
System.out.println(p.getAge());   // 25

p.setAge(-10);   // 무시
```

## 핵심 정리

* 잘못된 값 입력 방지 가능
* 안전하게 데이터 관리 가능

```java
car.start();
```


## 핵심 정리

* 내부는 숨기고 외부는 쉽게 사용
* 유지보수에 매우 좋습니다.

# 9. 문자열 비교

## 문제 상황

두 문자열이 같아 보이는데 왜 false가 나올까요?

## 예시 1

```java
String a = "java";
String b = "java";

System.out.println(a == b);   // true
```

문자열 상수풀 때문에 같은 주소를 참조할 수 있습니다.

---

## 예시 2

```java
String a = new String("java");
String b = new String("java");

System.out.println(a == b);        // false
System.out.println(a.equals(b));  // true
```

## 핵심 정리

* `==` : 주소 비교
* `.equals()` : 내용 비교

문자열 비교는 거의 항상 `.equals()` 사용합니다.
