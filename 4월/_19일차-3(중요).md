## 프로그래밍 언어 - Java(`JAVA02/Ch05`) 상속(Inheritance), 오버라이딩, 형변환(Casting) 심화 정리

## 오늘자 주요 태그

* **상속(Inheritance)**: 기존 클래스의 속성과 기능을 물려받아 새로운 클래스를 만드는 문법
* **생성자 호출 순서**: 자식 객체 생성 시 부모 생성자가 먼저 실행됨 (`super()`)
* **메서드 오버라이딩**: 부모 메서드를 자식 클래스에서 재정의
* **오버로딩 vs 오버라이딩**: 이름은 같지만 목적과 조건이 다름
* **업캐스팅 / 다운캐스팅**: 부모-자식 간 참조형 형변환
* **캡슐화 복습**: `private` + getter/setter 활용

## 오늘 배운 것

* **중요**: 상속은 코드 재사용성과 유지보수성을 높이기 위한 핵심 객체지향 문법이에요.
* **알아두기**: 자식 클래스 객체를 만들면 부모 생성자가 먼저 실행돼요.
* **핵심**: 업캐스팅은 자동 형변환, 다운캐스팅은 강제 형변환이에요.
* **실무 개념**: 다형성(Polymorphism)은 오버라이딩 + 업캐스팅과 깊은 관련이 있어요.

### 2. [Ch05] 캡슐화 + getter/setter 복습

#### Student 클래스 작성 (C001Ex.java)
* 필드를 private으로 보호하고 외부에서는 메서드로 접근해요.
* 정보 은닉(캡슐화)의 대표 예제예요.

```java
class StudentEx2 {
    private String name;
    private int id;

    void setName(String name){
        this.name = name;
    }

    String getName(){
        return this.name + " 학과";
    }

    void setId(int id){
        this.id = id;
    }

    int getId(){
        return this.id;
    }
}
```
#### main 실행 코드
```java
StudentEx2 student1 = new StudentEx2();

student1.setName("컴퓨터공학");
student1.setId(2026001);

System.out.println(
    "학생1의 학과 :" + student1.getName() +
    ", 학번 :" + student1.getId()
);
```
#### 예상 출력

```java
학생1의 학과 :컴퓨터공학 학과, 학번 :2026001
```
#### 핵심 정리
* 직접 접근 불가 → private
* 메서드로만 접근 → 데이터 보호 가능

### 3. [Ch05] 부모 생성자와 자식 생성자

#### 생성자 호출 순서 (C02InheritanceMain.java)
* 자식 객체를 만들면 부모 생성자가 먼저 실행돼요.
* super()는 부모 생성자를 호출하는 예약어예요.

```java
Point3D ob = new Point3D(10,20,30);
System.out.println(ob);
```
#### 실제 클래스 구조
```java
class Point3D extends Point2D {
    int z;

    Point3D(int x, int y, int z){
        super(x,y);
        this.z = z;
        System.out.println("Point3D(int x, int y, int z) 생성자 호출!");
    }
}
```
#### 예상 출력
```java
Point2D(int x, int y) 생성자 호출!
Point3D(int x, int y, int z) 생성자 호출!
Point3D [z=30, x=10, y=20]
```
#### 핵심 정리
* 부모 생성자가 먼저 실행됨
* super(x,y) 로 부모 생성자 호출

#### 4. [Ch05] 메서드 오버라이딩(Method Overriding)

#### 부모 기능을 자식이 재정의 (C03MethodOverriding.java)
* 메서드 이름, 매개변수, 반환형이 같아야 해요.
* 내용(body)만 다시 작성합니다.
  
#### 실제 코드
```java
class Animal {
    void sound() {
        System.out.println("소리낸다!");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("멍멍 소리낸다!");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("야옹 소리낸다!!");
    }
}
```
#### 실행 코드
``` java
Dog tori = new Dog();
tori.sound();

Cat navi = new Cat();
navi.sound();
```
#### 예상 출력
``` java
멍멍 소리낸다!
---------------
야옹 소리낸다!!
```
#### 핵심 정리
* 부모의 같은 메서드를 자식이 새롭게 구현
* 객체마다 다른 결과 출력 가능

### 5. [Ch05] 오버로딩 vs 오버라이딩

#### 차이점 정리 (C04OverLoadRiddingMain.java)

| 구분    | 오버로딩  | 오버라이딩 |
| ----- | ----- | ----- |
| 상속 필요 | X     | O     |
| 함수명   | 동일    | 동일    |
| 매개변수  | 달라야 함 | 동일    |
| 목적    | 편의성   | 다형성   |

#### 실제 코드 예시 (C04OverLoadRiddingMain.java)
```java
class Super {
    //오버로딩
    int sum(int x, int y){
        return x+y;
    }
    //오버로딩
    int sum(int x, int y, int z){
        return x+y+z;
    }
}

class Sub extends Super {
    //오버라이딩
    int sum(int x, int y){
        return x+y+1;
    }
    //오버로딩
    int sum(int x, double y){
        return (int)(x+y);
    }
}
```
#### ↑ 오버로딩,오버라이딩 예시

### 6. [Ch05] 상속 실습 문제 - ColorTV

#### TV를 상속받은 ColorTV 작성 (C05Ex.java)
* 부모 TV 클래스의 브랜드, 연도, 크기를 상속받아요.
* 자식 클래스는 색상(Color)을 추가했어요.

#### 실제 코드 (C05Ex.java)
```java
ColorTV myTV = new ColorTV("삼성",2026,32,"검정");
myTV.printProperty();
```
#### 클래스 구조
```java
class ColorTV extends TV{
    String Color;

    public ColorTV(String brand, int year, int inch, String Color) {
        super(brand, year, inch);
        this.Color = Color;
    }

    public void printProperty() {
        System.out.println(
            this.brand + " " +
            this.year + "년형 " +
            this.inch + "인치 " +
            this.Color
        );
    }
}
```
#### 예상 출력
```java
삼성 2026년형 32인치 검정
```
#### 핵심 정리
* 부모 TV 정보 재사용
* 자식 ColorTV는 색상만 추가

### 7. [Ch05] 업캐스팅 / 다운캐스팅

#### 실제 코드 (C06UpDownCastingMain.java)
```java
Super ob2 = new Sub();   // 업캐스팅
ob2.n1 = 10;

Sub down = (Sub)ob2;     // 다운캐스팅
down.n2 = 20;
```
#### 클래스 구조
```java
class Super{
    int n1;
}

class Sub extends Super{
    int n2;
}
```
#### 핵심 설명
**업캐스팅**
```java
Super ob = new Sub();
```
* 자식 객체를 부모 타입에 저장
* 자동 형변환
**다운캐스팅**
```java
Sub down = (Sub)ob;
```
* 다시 자식 타입으로 복원
* 강제 형변환 필요

## 오늘 핵심 정리
* 상속은 부모 코드를 재사용하는 핵심 문법이에요.
* extends 로 상속하고 super() 로 부모 생성자를 호출해요.
* 오버라이딩은 자식만의 기능 변경이에요.
* 업캐스팅은 다형성의 시작이에요.
* 다운캐스팅은 필요한 경우에만 안전하게 사용해야 해요.
