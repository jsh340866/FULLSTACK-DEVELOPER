## 프로그래밍 언어 - Java static 멤버와 싱글톤(Singleton) 패턴 심화 정리

## 오늘자 주요 태그

* **static 변수 & 메서드**: 공유 메모리(Class/Method 영역) 적재 / 객체 생성과는 무관하게 할당
* **싱글톤 패턴**: 객체 생성을 제한하여 단 하나의 인스턴스만 보장하는 설계 기법
* **메모리 구조**: Stack, Heap, 그리고 static이 저장되는 Class Area의 관계 이해
* **유틸리티 클래스**: 공통 기능을 static 메서드로 묶어 객체 생성 없이 클래스명으로 접근

## 오늘 배운 것

* **중요**: `static` 키워드는 객체마다 독립적인 공간을 갖는 것이 아니라, 공유 메모리에 적재되어 모든 객체가 동일한 저장 공간을 공유해요.
* **알아두기**: `static` 메서드 안에서는 인스턴스 변수를 사용할 수 없으며, `static` 변수나 지역 변수만 사용 가능해요.
* **핵심**: 싱글톤 패턴은 생성자를 `private`으로 선언해 외부 생성을 막고, `getInstance()` 메서드를 통해서만 객체를 반환하는 것이 핵심이에요.

### 1. [Ch04] static 멤버의 특징과 활용

#### static 변수와 인스턴스 변수의 차이 (C01StaticMain.java)
* `static` 변수는 클래스당 하나만 생성되어 모든 객체가 동일한 값을 참조해요.
* 실습 코드에서 `ob1.n1 = 2323;`으로 값을 바꾸면, 출력 시 `ob2`의 `n1` 값도 동일하게 `2323`으로 변경되는 것을 확인할 수 있었어요.



```java
class C01Simple {
    static int n1; // 공유 변수: 모든 객체가 이 공간을 함께 써요
    int n2;        // 인스턴스 변수: 객체마다 자기만의 공간을 가져요
}

// 객체 생성 없이도 클래스명으로 직접 접근 가능!
C01Simple.n1 = 1234;
```

#### static 메서드 제약 사항 (C02StaticMain.java)
* `static` 메서드는 객체가 생성되기 전부터 메모리에 존재하므로, 생성 시점을 알 수 없는 일반 멤버 변수(`n2`)에는 접근할 수 없어요.
```Java
static void func2() {
    n1 = 300; // static 변수는 사용 가능
    // n2 = 400; // 에러 발생! 일반 멤버는 언제 만들어질지 알 수 없기 때문이에요
}
```

### 2. [Ch04] 싱글톤 패턴 (Singleton Pattern)
#### 객체 생성을 제한하는 설계 (C03SingleTonMain.java)
  * 프로그램 전체에서 단 하나의 인스턴스만 존재하도록 보장하여 메모리 효율을 높여요.

```Java
// 싱글톤 기본 공식
class Company {
    // 1. 클래스 내부에서만 쓸 수 있는 유일한 인스턴스 변수 생성
    private static Company instance; 
    
    // 2. 외부에서 new를 쓰지 못하게 생성자를 숨겨요
    private Company() {} 
    
    // 3. 외부에서는 오직 이 메서드로만 객체를 받아갈 수 있어요
    public static Company getInstance() {
        if(instance == null) // 아직 객체가 없다면 그때 하나만 딱 만들어요
            instance = new Company();
        return instance; // 이미 있다면 원래 있던 걸 돌려줘요
    }
}
```

### 3. [Ch04] 실습 및 응용
#### static 유틸리티 클래스 작성 (C04Ex.java)
* 별도의 데이터 저장 없이 기능만 제공할 때는 모든 메서드를 `static`으로 선언해요.
* `System.arraycopy(a, 0, c, 0, a.length);`를 사용하여 반복문 없이도 배열을 효율적으로 복사하는 로직을 실습했어요.

#### 싱글톤 패턴 실전 연습 (C05Ex.java)
* `ShopService` 클래스에 싱글톤을 적용하고, `obj1 == obj2` 비교를 통해 두 변수가 가리키는 주소값이 정확히 일치함을 증명했어요

```Java
// C05Ex.java 실행 결과
ShopService obj1 = ShopService.getInstance();
ShopService obj2 = ShopService.getInstance();

if (obj1 == obj2) {
    System.out.println("같은 ShopService 객체입니다."); // 이 문장이 출력돼요!
}
```
