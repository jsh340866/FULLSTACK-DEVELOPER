## 오늘 배운 것

    MySQL cmd 명령어 참조

```
    *** MySQL cmd 명렁어 ***
    - 대소문자 구분을 하지않음
    - 명령어가 끝날때 세미콜론(';') 달아줄 것
    
    1. 데이터베이스 생성 및 선택
    CREATE DATABASE test1db;  -- 'test1db'라는 이름의 데이터베이스를 생성해요.
    USE test1db;             -- "지금부터 이 데이터베이스를 사용할게!"라고 선택해요.


    2-A. 데이터베이스 생성 
    CREATE TABLE tbl_user (
      userId VARCHAR(45) PRIMARY KEY,  -- 아이디를 만들면서 바로 기본키로 지정!
      userName VARCHAR(45) NULL,
      password VARCHAR(255) NULL
    );

    2-B 데이터베이스 생성 
    CREATE TABLE tbl_user (
      userId VARCHAR(45) NOT NULL,
      userName VARCHAR(45) NULL,
      password VARCHAR(255) NULL,
      PRIMARY KEY (userId)             -- 모든 컬럼을 적고 난 뒤 기본키를 지정!
    );


    3. 구조 확인 및 수정 (CMD 활용)
    SHOW DATABASES;: 현재 내 컴퓨터에 만들어진 모든 데이터베이스 목록을 확인해요.
    DESC tbl_user;: 생성한 테이블의 설계도를 한눈에 확인해요. (컬럼명, 타입 등)
    
    ALTER TABLE ... ADD COLUMN ... AFTER: 테이블 열(colunm) 추가
    ALTER TABLE ... DROP COLUMN ... AFTER: 테이블 열(colunm) 삭제
    ALTER TABLE ... CHANGE COLUMN ... AFTER: 테이블 열(colunm) 수정
        예시)
        - ALTER TABLE tbl_user ADD COLUMN phone VARCHAR(45) NULL AFTER userName;
          이 명령어는 userName 뒤에 phone 칸을 쏙 집어넣어 줘요.

    4. 데이터 삽입 (INSERT INTO)
    INSERT INTO `test1db`.`tbl_user` (`userId`, `userName`, `password`) 
    VALUES ('user1', '홍길동', '1234');

    5. 데이터 조회 (SELECT)
    SELECT * FROM tbl_user;
        - * (애스터리스크): "모든 컬럼을 다 보여줘!"라는 뜻이에요. (userId, userName, password 전부 다)

    6. 데이터 수정 (UPDATE)
    UPDATE `test1db`.`tbl_user`SET `userName` = '티모ㅇ', `password` = '12341252' WHERE (`userId` = 'user1');
        - WHERE 조건: 가장 중요한 부분이에요. "누구의 데이터를 바꿀 것인가"를 정해요.

    7. 데이터 삭제 (DELETE)
    DELETE FROM `test1db`.`tbl_user` WHERE (`userId` = 'user1');
        - WHERE 조건: 가장 중요한 부분이에요. "누구의 데이터를 삭제할 것인가"를 정해요.
