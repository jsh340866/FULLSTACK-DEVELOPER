## 프로그래밍 언어 - Java(`JAVA02/Ch05`) 업캐스팅 / 다운캐스팅 / 다형성 심화 정리

## 오늘자 주요 태그

* **업캐스팅(UpCasting)**: 자식 객체를 부모 타입 참조변수에 저장하는 자동 형변환
* **다운캐스팅(DownCasting)**: 부모 타입 참조변수를 다시 자식 타입으로 변환
* **다형성(Polymorphism)**: 같은 부모 타입으로 여러 자식 객체를 다루는 개념
* **오버라이딩(Overriding)**: 부모 메서드를 자식 클래스에서 재정의
* **instanceof**: 객체 타입 확인 후 안전한 형변환 수행
* **인터페이스 형변환**: 클래스뿐 아니라 인터페이스 타입으로도 업캐스팅 가능

## 오늘 배운 것

* **중요**: 업캐스팅된 참조변수는 부모가 가진 멤버만 접근 가능해요.
* **핵심**: 오버라이딩된 메서드는 업캐스팅 상태에서도 자식 메서드가 실행돼요.
* **주의**: 다운캐스팅은 실제 객체가 해당 자식 타입일 때만 가능해요.
* **실무 개념**: 다형성은 유지보수성과 확장성을 높이는 핵심 객체지향 기술이에요.

### 1. [Ch05] 기본 업캐스팅과 다운캐스팅

#### Dog 객체를 Animal 타입으로 저장 (C09Ex01BasicUpCasting.java)

* 자식 객체를 부모 타입으로 저장하면 자동 형변환돼요.
* 부모 타입에서는 자식 전용 메서드 호출이 불가능해요.
* 다시 자식 타입으로 다운캐스팅하면 접근 가능해져요.

```java
Dog d = new Dog();
Animal da = d;

da.makeSound();

Dog down = (Dog)da;
down.fetch();
```

#### 코드 설명
* `Dog D = new Dog();`
    * → Dog 객체 생성
* `Animal DA = D;`
    * → Dog 객체를 Animal 타입으로 저장 (업캐스팅)
* `DA.makeSound();`
    * → Animal 타입 변수지만 실제 객체는 Dog 이므로 Dog의 메서드 실행
* `Dog down = (Dog)DA;`
    * → 다시 Dog 타입으로 다운캐스팅
* `down.fetch();`
    * → Dog 전용 기능 사용 가능
      
#### 예상 출력
```java
멍멍!
강아지가 공을 가져옵니다!
강아지가 공을 가져옵니다!
```

#### 핵심 정리

* 업캐스팅 후 자식 전용 메서드는 바로 사용 불가
* 다운캐스팅하면 다시 사용 가능

### 2. [Ch05] 업캐스팅 상태의 오버라이딩

#### 실제 코드 (C10Ex02MethodAccess.java)

```java
Car car1 = new Car();
Vehicle carV = car1;

carV.start();

((Car)carV).turboBoost();

Car downCar = (Car)carV;
downCar.turboBoost();
```

#### 코드 설명
* `Vehicle carV = car1;`
    * → Car 객체를 Vehicle 타입으로 업캐스팅
* `carV.start();`
    * → Vehicle 타입 변수지만 Car의 `start()` 실행됨
* 이유:
    * Car 클래스가 부모의 `start()`를 재정의했기 때문이에요.
 
```java
class Vehicle {
    void start() {
        System.out.println("탈것의 시동을 겁니다");
    }
}

class Car extends Vehicle {
    void start() {
        System.out.println("자동차의 시동을 겁니다");
    }
}
```

#### 예상 출력

```java
자동차의 시동을 겁니다
자동차가 터보로 가속합니다!
자동차가 터보로 가속합니다!
```

#### 핵심 정리

* 참조형은 부모여도 실제 객체가 자식이면 자식 메서드 실행
* 이것이 다형성 핵심이에요.

### 3. [Ch05] 다중 상속 구조 형변환

#### 실제 코드 (C11Ex03MultiLevel.java)
```java
Shape c1 = new ColoredCircle();

c1.draw();

Circle c2 = (Circle)c1;
c2.calculateArea();

ColoredCircle c3 = (ColoredCircle)c2;
c3.applyColor();
```

#### 클래스 구조
```java
Shape
 └── Circle
      └── ColoredCircle
```

#### 코드 설명

* `Shape c1 = new ColoredCircle();`
    * → 가장 상위 부모 타입으로 저장 가능
* `c1.draw();`
    * → 실제 객체는 ColoredCircle 이므로 재정의된 draw 실행
* `Circle c2 = (Circle)c1;`
    * → 중간 부모 타입으로 다운캐스팅
* `ColoredCircle c3 = (ColoredCircle)c2;`
    * → 원래 자식 타입으로 복귀
 
#### 예상 출력

```java
색칠된 원을 그립니다
원의 넓이를 계산합니다
원에 색을 칠합니다
```

#### 핵심 정리

* 조상 타입으로도 저장 가능
* 필요한 기능에 따라 단계별 다운캐스팅 가능

### 4. [Ch05] instanceof 활용

#### 실제 코드 (C07UpDownCastingMain.java)

```java
if(obj instanceof C) {
    C down = (C)obj;
    down.c = 300;
}

if(obj instanceof F) {
    F down = (F)obj;
    down.f = 700;
}
```

#### 실행 코드

```java
UpDownCastingTestFunc(new F());
```

#### 결과 분석

```java
F 객체는 아래 구조예요.

A
└── C
    └── F
```

그래서:

* A 멤버 사용 가능
* C 멤버 사용 가능
* F 멤버 사용 가능
 
#### 예상 출력

```java
F [f=700, c=300, a=100]
```

#### 핵심 정리

* 타입 확인 없이 다운캐스팅하면 오류 발생 가능
* instanceof로 먼저 검사해야 안전해요.

### 5. [Ch05] 인터페이스 업캐스팅

#### 실제 코드 (C12Ex04InterfaceImpl.java)

```java
Playable p = new Piano();
p.play();
```

#### 클래스 구조

```java
interface Playable
      ↑
 Instrument
      ↑
   Piano
```

#### 내부 코드

```java
class Piano extends Instrument {
    public void play() {
        System.out.println("피아노를 연주합니다");
    }

    void tune() {
        System.out.println("피아노를 조율합니다");
    }
}
```

#### 설명

* `Playable p = new Piano();`
    * → Piano 객체를 인터페이스 타입으로 저장
* `p.play();`
    * → Piano의 play 실행
* 단, `p.tune();` 은 불가능
  
#### 예상 출력

```java
피아노를 연주합니다
```

#### 핵심 정리

* 인터페이스 타입으로 객체를 통일 관리 가능
* 실무에서 가장 많이 사용하는 구조예요.

### 6. [Ch05] 복합 계층 구조 다형성

#### 실제 코드 (C13Ex05ComplexHierarchy.java)

```java
Duck d = new Duck();

Bird b = d;
Flyable f = d;
Swimmable s = d;

b.fly();
f.fly();
s.swim();

((Duck)b).quack();
```

#### 클래스 구조

```java
Flyable
   ↑
 Bird
   ↑
 Duck ── implements Swimmable
```

#### 예상 출력

```java
오리가 날아갑니다
오리가 날아갑니다
오리가 수영합니다
오리가 꽥꽥거립니다
```

#### 핵심 정리

* 하나의 Duck 객체를 여러 타입으로 참조 가능
* 이것이 객체지향 유연성의 핵심이에요.

## 오늘 핵심 정리
* 업캐스팅은 자식 객체를 부모 타입으로 저장하는 자동 형변환이에요.
* 업캐스팅 상태에서도 오버라이딩 메서드는 자식 것이 실행돼요.
*  자식 전용 기능은 다운캐스팅 후 사용할 수 있어요.
*  `instanceof`는 안전한 형변환을 위해 꼭 필요해요.
*  인터페이스 업캐스팅은 실무에서 매우 중요해요.
