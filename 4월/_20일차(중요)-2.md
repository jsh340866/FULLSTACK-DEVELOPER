## 오늘 배운 것

오늘은 Java 객체지향 설계에서 중요한 **추상클래스**, **인터페이스**, **다형성**를 학습했습니다.

즉, “클래스를 만드는 법”이 아니라 **유지보수 잘 되는 프로그램 구조**를 배우는 단계입니다.

---

### 1. 추상클래스(Abstract Class)

#### 설명

추상클래스는 **공통 기능은 미리 만들고**, 세부 기능은 자식 클래스에게 맡기는 클래스입니다.

파일 코드:

```java
abstract class Parent2 {
    abstract void func();
}
```
여기서 `func()`는 선언만 있고 내용이 없습니다.
그래서 `Parent2`를 상속받는 자식은 반드시 `func()`를 구현해야 합니다.

#### 실제 코드 인용

```java
class Son2 extends Parent2 {

    @Override
    void func() {
        System.out.println("Son2's 에서 재정의한 func...");
    }
}
```
```java
Parent2 ob6 = new Son2();
ob6.func();
```

#### 왜 이렇게 쓰는가?
`Parent2`는 “func 기능이 필요하다”는 규칙만 만들고,
실제 동작은 `Son2`가 담당합니다.

즉,
* 부모 = 설계도
* 자식 = 실제 구현체

#### 예상 출력
```java
Son2's 에서 재정의한 func...
```

#### 핵심 정리

* 추상클래스는 `new Parent2()` 불가능
* 자식이 추상 메서드 구현해야 함
* 부모 타입으로 자식 객체 관리 가능

---

### 2. 추상클래스 실전 예제 - 단위 변환기
#### 추상클래스 예제
```java
abstract class Converter {
    abstract protected double convert(double src);
    abstract protected String getSrcString();
    abstract protected String getDestString();
}
```

이 클래스는:

* 무엇을 입력받는지
* 무엇으로 바꾸는지
* 계산은 어떻게 하는지
를 자식에게 맡깁니다.

#### 원 → 달러 변환 코드
```java
class Won2Dollar extends Converter {
    protected double convert(double src) {
        return src / ratio;
    }
}
```
24000원을 입력하면:
```java
24000 / 1450 = 약 16.55달러
```

#### 섭씨 → 화씨 코드
```java
class 섭씨to화씨 extends Converter {
    protected double convert(double src) {
        return (src * 1.8) + 32;
    }
}
```
30도 입력 시:
```java
(30 × 1.8) + 32 = 86.0
```
핵심 정리

같은 `Converter`를 상속받았지만 계산 방식은 완전히 다릅니다.

이것이 **추상화 + 다형성** 입니다.

### 3. 인터페이스(Interface)
설명

인터페이스는 “이 기능은 반드시 있어야 한다”는 **규칙서**입니다.

파일 코드:
```java
interface Remocon {
    void powerOn();
    void powerOff();
    void setVolumn(int vol);
}
```
리모컨 기능을 가진 제품이라면 반드시:

* 전원 켜기
* 전원 끄기
* 볼륨 조절
이 있어야 한다는 뜻입니다.

---

### 4. TV / Radio 구현 예제

#### 실제 코드
```java
class Tv implements Remocon {

    public void powerOn() {
        System.out.println("Tv를 켭니다");
    }

    public void powerOff() {
        System.out.println("Tv를 끕니다");
    }
}
class Radio implements Remocon {

    public void powerOn() {
        System.out.println("Radio를 켭니다");
    }
}
```
#### 같은 명령, 다른 결과
```java
Tv tv = new Tv();
Radio radio = new Radio();

tv.powerOn();
radio.powerOn();
```
출력:
```java
Tv를 켭니다
Radio를 켭니다
```
같은 `powerOn()`인데 결과가 다릅니다.

이게 **다형성** 입니다.

---

### 5. 업캐스팅 + 다형성

#### 파일 코드
```java
public static void TurnOn(Remocon remocon) {
    remocon.powerOn();
}
```
매개변수를 `Tv`, `Radio`로 받지 않고 `Remocon`으로 받았습니다.

#### 사용 코드
```java
TurnOn(tv);
TurnOn(radio);
TurnOn(smartTv);
```

#### 왜 좋은가?

새 제품 `Speaker`가 추가되어도:
```java
class Speaker implements Remocon
```
이렇게 만들면 `TurnOn()` 수정 없이 그대로 사용 가능합니다.

#### 핵심 정리
부모 타입(인터페이스) 하나로 모든 자식 객체 제어 가능.

---

### 6. 다중 인터페이스 구현

#### 파일 코드
```java
interface Browser {
    void searchURL(String url);
}
```
```java
class SmartTv extends Tv implements Browser {

    public void searchURL(String url) {
        System.out.println(url + " 로 이동합니다.");
    }
}
```

#### 설명

SmartTv는:

* TV 기능
* 리모컨 기능
* 인터넷 기능
모두 가집니다.

#### 실행 결과
```java
smartTv.searchURL("https://exam_all");
```
```java
https://exam_all 로 이동합니다.
```

### 7. 타이어 교체 구조 (의존성 분리)

#### 파일 코드
```java
interface Tire {
    void run();
}
```
```java
car.FR = new 금호타이어();
car.BL = new 금호타이어();
```
#### 설명
자동차는 Tire 규격만 알면 됩니다.

즉:

* 한국타이어
* 금호타이어
* 넥센타이어

무엇이든 장착 가능.

자동차 입장에서는 브랜드를 몰라도 됩니다.

#### 핵심 정리
결합도 낮은 구조 = 유지보수 쉬움

### 8. 프린터 예제

#### 실제 코드
```java
Printer p = new Inkjet();
p.print();

p = new Laser();
p.print();
```

#### 출력
```java
Inkjet 프린터 출력
Laser 프린터 출력
```

#### 설명

변수는 `Printer p` 하나인데 객체만 교체했습니다.

즉, 사용하는 코드는 그대로 두고 기능만 변경 가능.
