```
    *** MySQL DDL DML DCL TCL 정리***

    DDL(Data Definition Language): CREATE, ALTER, DROP, TRUNCATE(초기화)
        - MySQL 기준 ddl명령어 사용시 autocommit 작동
    DML(Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
        - 기본적인 테이블 명령어
    DCL(Data Control Language): GRANT, REVOKE
        - 사용자 권한 부여 or 삭제
    TCL(Transaction Control Language): COMMIT, ROLLBACK, CHECKPOINT
        - 커밋,롤백 등
    
```
```
    *** MySQL cmd 명령어

    1. GRANT (권한 부여)
    GRANT ... ON ... TO... ;	// all privileges : 모든 권한 명령어
    grant all privileges on test2db.* to user1@localhost;
        - all privileges : 모든 권한 명령어
        - ON test2db.*: "test2db라는 데이터베이스 안에 있는 모든 테이블(*)에 대해서만!"
        - TO user1@localhost: "user1이라는 친구한테 줄 건데, 이 친구는 꼭 내 컴퓨터(localhost)로 접속했을 때만

    2. REVOKE (권한 회수)
    REVOKE ... ON ... FROM...;
    revoke insert on test2db.* from user30@localhost;
        - REVOKE INSERT: "데이터를 새로 넣는(INSERT) 권한만 딱 뺏을게!"
        - FROM user30@localhost: "user30 친구한테서 뺏어올 거야."

    3. DROP USER (계정 삭제)
    DROP USER ... ;
    drop user dev_user@'%';
        - dev_user@'%': 여기서 '%'는 "어느 컴퓨터(외부 IP)에서 접속하든 상관없이"라는 뜻이에요
        - dev_user라는 계정 자체가 사라져서 아예 로그인을 할 수 없게 된답니다.


```    
```
    *** TRANSACTION(트랜색션) ***
    SET autocommit = 0;
	
	START TRANSACTION;
		INSERT INTO tbl_user VALUES('aaa','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('bbb','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('ccc','홍길동','1234'); -- 
	    SAVEPOINT sp1;
		INSERT INTO tbl_user VALUES('ddd','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('eee','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('fff','홍길동','1234'); -- 
	    SAVEPOINT sp2;
	    INSERT INTO tbl_user VALUES('ggg','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('hhh','홍길동','1234'); -- 
		INSERT INTO tbl_user VALUES('iii','홍길동','1234'); -- 
	    SAVEPOINT sp3;   
		ROLLBACK TO sp2;
	COMMIT;
	- 출력으로 sp2 위로 적용된다 

```
```
	*** 백업 / 복구 ***

	1. 데이터 백업 (내보내기: Export)
		1) 데이터베이스 전체 백업
			- mysqldump -u root -p1234 world > world_backup.sql
			- 해석: world라는 데이터베이스의 통째로(구조+데이터) world_backup.sql 파일에 담아줘!

		2) 특정 테이블만 백업
			- mysqldump -u root -p1234 world city > city_backup.sql
			- 해석: world DB 안에 있는 city 테이블 하나만 쏙 골라서 백업해줘! (용량이 클 때 유용해요.)

		3) 모든 데이터베이스 백업
			- mysqldump -u root -p1234 --all-databases > all_backup.sql
			- 해석: 이 서버에 있는 모든 데이터베이스를 하나도 빠짐없이 전부 다 파일 하나에 백업해줘!


	2. 데이터 복구 (가져오기: Import)
		1) MySQL 접속: mysql -u root -p1234

		2) 스크립트 실행: source all_backup.sql
			- 해석: "아까 만들어둔 all_backup.sql이라는 대본(스크립트)을 처음부터 끝까지 쭉 읽어서 실행해줘!"
			- 이 명령어가 끝나면 파일에 적혀있던 대로 테이블이 다시 만들어지고 데이터가 차곡차곡 쌓이게 돼요.
