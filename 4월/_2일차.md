```
    *** MySQL DDL DML DCL TCL 정리***

    DDL(Data Definition Language): CREATE, ALTER, DROP, TRUNCATE(초기화)
        - MySQL 기준 ddl명령어 사용시 autocommit 작동
    DML(Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
        - 기본적인 테이블 명령어
    DCL(Data Control Language): GRANT, REVOKE
        - 사용자 권한 부여 or 삭제
    TCL(Transaction Control Language): COMMIT, ROLLBACK, CHECKPOINT
        - 커밋,롤백 등
    
```
```
    *** MySQL cmd 명령어

    1. GRANT (권한 부여)
    GRANT ... ON ... TO... ;	// all privileges : 모든 권한 명령어
    grant all privileges on test2db.* to user1@localhost;
        - all privileges : 모든 권한 명령어
        - ON test2db.*: "test2db라는 데이터베이스 안에 있는 모든 테이블(*)에 대해서만!"
        - TO user1@localhost: "user1이라는 친구한테 줄 건데, 이 친구는 꼭 내 컴퓨터(localhost)로 접속했을 때만

    2. REVOKE (권한 회수)
    REVOKE ... ON ... FROM...;
    revoke insert on test2db.* from user30@localhost;
        - REVOKE INSERT: "데이터를 새로 넣는(INSERT) 권한만 딱 뺏을게!"
        - FROM user30@localhost: "user30 친구한테서 뺏어올 거야."

    3. DROP USER (계정 삭제)
    DROP USER ... ;
    drop user dev_user@'%';
        - dev_user@'%': 여기서 '%'는 "어느 컴퓨터(외부 IP)에서 접속하든 상관없이"라는 뜻이에요
        - dev_user라는 계정 자체가 사라져서 아예 로그인을 할 수 없게 된답니다.


```
