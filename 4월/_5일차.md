### SELECT 요점정리 

#### 1. 기본 SELECT

- **전체 조회**
  - `SELECT * FROM usertbl;`
  - `SELECT * FROM buytbl;`

- **특정 컬럼만 조회**
  - `SELECT userid, birthYear FROM usertbl;`

- **별칭(alias)**
  - `SELECT userid AS '아이디', birthYear AS '생년월일' FROM usertbl;`

- **문자열 결합(concat)**
  - `SELECT concat(mobile1,'-',mobile2) AS '연락처' FROM usertbl;`


#### 2. WHERE 조건절 - 비교 연산자

- **동등 비교(=)**
  - `SELECT * FROM usertbl WHERE name='김경호';`
  - `SELECT * FROM usertbl WHERE userId='LSG';`

- **대소 비교(>=, <=)**
  - `SELECT * FROM usertbl WHERE birthYear >= 1970;`
  - `SELECT * FROM usertbl WHERE height <= 170;`


#### 3. WHERE 조건절 - 논리 연산자

- **AND / OR**
  - `SELECT * FROM usertbl WHERE birthYear >= 1970 AND height >= 180;`
  - `SELECT * FROM usertbl WHERE birthYear >= 1970 OR height >= 180;`

- **범위 조건: BETWEEN**
  - `WHERE height BETWEEN 170 AND 180;`
  - 위 문장은 `height >= 170 AND height <= 180` 과 동일

- **집합 조건: IN**
  - `WHERE addr IN ('서울','경기');`
  - 위 문장은 `addr='서울' OR addr='경기'` 와 동일

- **패턴 조건: LIKE**
  - `LIKE '김%'` : "김"으로 시작
  - `LIKE '%수'` : "수"로 끝
  - `LIKE '%경%'` : "경" 포함
  - `LIKE '김__'` : "김" + 임의 2글자(총 3글자)


#### 4. buytbl 실습 문제에서 나온 패턴

- **구매수량(amount) 5 이상**
  - `SELECT * FROM buytbl WHERE amount >= 5;`

- **가격(price) 50~500, userid/prodName만 출력**
  - `SELECT userid, prodName FROM buytbl WHERE price BETWEEN 50 AND 500;`

- **구매수량(amount) 10 이상 OR 가격 조건**
  - SQL 파일 예시: `WHERE amount >= 10 OR price <= 100;`
  - 문제 문장 기준이 “가격 100 이상”이면 보통 `price >= 100` 형태로 작성

- **UserId가 K로 시작**
  - `SELECT * FROM buytbl WHERE userid LIKE 'K%';`

- **groupName이 '서적' 또는 '전자'**
  - `SELECT * FROM buytbl WHERE groupName IN ('전자','서적');`

- **prodName이 책 OR userid가 W로 끝**
  - `SELECT * FROM buytbl WHERE prodName IN ('책') OR userid LIKE '%W';`


#### 5. 빈 값/NULL 처리

- **빈 문자열 제외**
  - `WHERE groupName != '';`
  - `WHERE groupName <> '';`

- **NULL 제외**
  - `WHERE groupName IS NOT NULL;`

> 참고: `LIKE`는 리스트가 아니라 “패턴 1개”로 검색하는 문법이라  
> `addr LIKE ('서울','경기')` 보다는 `addr IN ('서울','경기')` 형태가 일반적입니다.
