### 정규화

#### 1. 정규화가 하는 일
- 테이블 설계에서 **중복·이상현상(삽입/갱신/삭제 이상)** 을 줄이기 위해, **함수적 종속**을 기준으로 테이블을 나눔.
- 단계: **1NF → 2NF → 3NF → BCNF → (4NF …)**.
 
#### 2. 1NF (제1정규형)
- **한 칸에는 원자값(atomic value)만** — 콤마로 묶인 값·리스트 금지.
- CSV 예: `상품목록`의 `"사과, 바나나"` → **행 분리**로 한 행·한 사실.
- 수강 예: `"P1, P3"` / `"이교수, 홍교수"` → **교수 한 명당 한 행**.
  
#### 3. 2NF (제2정규형)
- 전제: **1NF**
- 복합 PK `(A, B)`에서 **A만으로 결정되는 비-PK 속성** → **부분 종속 제거**(다른 테이블로 분리).
- CSV 예: `(학번, 과목코드)`인데 `학과명·학과사무실`이 **학번만**으로 결정 → 학생(학과) / 수강(성적) 분리.
- 주문 예: PK `(주문ID, 상품코드)`인데 `고객명·도시`가 **주문/고객 쪽**으로만 정해지면 → **고객 테이블** + 주문에는 `고객ID(FK)`.
  
#### 4. 3NF (제3정규형)
- 전제: **2NF**
- `A → B`, `B → C` 이고 ** B가 후보키가 아님** → **이행 종속** (`A → C`) 제거.
- CSV 예: 사원 → 부서코드 → 부서명·부서위치 → **사원** + **부서** 테이블 분리.
  
#### 5. BCNF
- **모든 결정자가 후보키**가 되도록 (결정자 규칙 강화).
- CSV 예: 학생·과목·담당교수 — 교수가 과목을 결정하는 식의 종속이 있으면 **담당/개설** 같은 테이블로 분해하는 방향.
  
#### 6. 4NF
- **다치 종속(MVD)** 제거 — 서로 **독립인** 여러 다중값을 한 테이블에 섞지 않음.
- CSV 예: 학생ID당 전화 여러 개·이메일 여러 개가 **곱하기처럼** 쌓임 → **학생–전화**, **학생–이메일** 테이블 분리.
  
#### 7. CSV 연습문제 흐름 (요약)
- **문제 1** (PK: `주문ID`, `상품코드`): 1NF에서 상품목록 분해 → 2NF에서 주문·고객·상품 분리 방향.
- **문제 2** (PK: `학번`, `과목코드`, `교수ID` 목표): 1NF에서 다중 교수 행 분해 → 2NF에서 성적 / 학생 / 과목–교수 분리.
  
#### 8. 한 줄 암기
- **1NF**: 원자값.
- **2NF**: 복합키 **부분 종속** 제거.
- **3NF**: **이행 종속** 제거.
- **BCNF**: **결정자 = 후보키**.
- **4NF**: 독립 다중값 **섞지 않기**.

### SELECT

#### 1. 기본 SELECT

- **전체 조회**
  - `SELECT * FROM usertbl;`
  - `SELECT * FROM buytbl;`

- **특정 컬럼만 조회**
  - `SELECT userid, birthYear FROM usertbl;`

- **별칭(alias)**
  - `SELECT userid AS '아이디', birthYear AS '생년월일' FROM usertbl;`

- **문자열 결합(concat)**
  - `SELECT concat(mobile1,'-',mobile2) AS '연락처' FROM usertbl;`


#### 2. WHERE 조건절 - 비교 연산자

- **동등 비교(=)**
  - `SELECT * FROM usertbl WHERE name='김경호';`
  - `SELECT * FROM usertbl WHERE userId='LSG';`

- **대소 비교(>=, <=)**
  - `SELECT * FROM usertbl WHERE birthYear >= 1970;`
  - `SELECT * FROM usertbl WHERE height <= 170;`


#### 3. WHERE 조건절 - 논리 연산자

- **AND / OR**
  - `SELECT * FROM usertbl WHERE birthYear >= 1970 AND height >= 180;`
  - `SELECT * FROM usertbl WHERE birthYear >= 1970 OR height >= 180;`

- **범위 조건: BETWEEN**
  - `WHERE height BETWEEN 170 AND 180;`
  - 위 문장은 `height >= 170 AND height <= 180` 과 동일

- **집합 조건: IN**
  - `WHERE addr IN ('서울','경기');`
  - 위 문장은 `addr='서울' OR addr='경기'` 와 동일

- **패턴 조건: LIKE**
  - `LIKE '김%'` : "김"으로 시작
  - `LIKE '%수'` : "수"로 끝
  - `LIKE '%경%'` : "경" 포함
  - `LIKE '김__'` : "김" + 임의 2글자(총 3글자)


#### 4. buytbl 실습 문제에서 나온 패턴

- **구매수량(amount) 5 이상**
  - `SELECT * FROM buytbl WHERE amount >= 5;`

- **가격(price) 50~500, userid/prodName만 출력**
  - `SELECT userid, prodName FROM buytbl WHERE price BETWEEN 50 AND 500;`

- **구매수량(amount) 10 이상 OR 가격 조건**
  - SQL 파일 예시: `WHERE amount >= 10 OR price <= 100;`
  - 문제 문장 기준이 “가격 100 이상”이면 보통 `price >= 100` 형태로 작성

- **UserId가 K로 시작**
  - `SELECT * FROM buytbl WHERE userid LIKE 'K%';`

- **groupName이 '서적' 또는 '전자'**
  - `SELECT * FROM buytbl WHERE groupName IN ('전자','서적');`

- **prodName이 책 OR userid가 W로 끝**
  - `SELECT * FROM buytbl WHERE prodName IN ('책') OR userid LIKE '%W';`


#### 5. 빈 값/NULL 처리

- **빈 문자열 제외**
  - `WHERE groupName != '';`
  - `WHERE groupName <> '';`

- **NULL 제외**
  - `WHERE groupName IS NOT NULL;`

> 참고: `LIKE`는 리스트가 아니라 “패턴 1개”로 검색하는 문법이라  
> `addr LIKE ('서울','경기')` 보다는 `addr IN ('서울','경기')` 형태가 일반적입니다.
