### 정규화

#### 1. 정규화가 하는 일
- 테이블 설계에서 **중복·이상현상(삽입/갱신/삭제 이상)** 을 줄이기 위해, **함수적 종속**을 기준으로 테이블을 나눔.
- 단계: **1NF → 2NF → 3NF → BCNF → (4NF …)**.
 
#### 2. 1NF (제1정규형)
- **한 칸에는 원자값(atomic value)만** — 콤마로 묶인 값·리스트 금지.
- CSV 예: `상품목록`의 `"사과, 바나나"` → **행 분리**로 한 행·한 사실.
- 수강 예: `"P1, P3"` / `"이교수, 홍교수"` → **교수 한 명당 한 행**.
  
#### 3. 2NF (제2정규형)
- 전제: **1NF**
- 복합 PK `(A, B)`에서 **A만으로 결정되는 비-PK 속성** → **부분 종속 제거**(다른 테이블로 분리).
- CSV 예: `(학번, 과목코드)`인데 `학과명·학과사무실`이 **학번만**으로 결정 → 학생(학과) / 수강(성적) 분리.
- 주문 예: PK `(주문ID, 상품코드)`인데 `고객명·도시`가 **주문/고객 쪽**으로만 정해지면 → **고객 테이블** + 주문에는 `고객ID(FK)`.
  
#### 4. 3NF (제3정규형)
- 전제: **2NF**
- `A → B`, `B → C` 이고 ** B가 후보키가 아님** → **이행 종속** (`A → C`) 제거.
- CSV 예: 사원 → 부서코드 → 부서명·부서위치 → **사원** + **부서** 테이블 분리.
  
#### 5. BCNF
- **모든 결정자가 후보키**가 되도록 (결정자 규칙 강화).
- CSV 예: 학생·과목·담당교수 — 교수가 과목을 결정하는 식의 종속이 있으면 **담당/개설** 같은 테이블로 분해하는 방향.
  
#### 6. 4NF
- **다치 종속(MVD)** 제거 — 서로 **독립인** 여러 다중값을 한 테이블에 섞지 않음.
- CSV 예: 학생ID당 전화 여러 개·이메일 여러 개가 **곱하기처럼** 쌓임 → **학생–전화**, **학생–이메일** 테이블 분리.
  
#### 7. CSV 연습문제 흐름 (요약)
- **문제 1** (PK: `주문ID`, `상품코드`): 1NF에서 상품목록 분해 → 2NF에서 주문·고객·상품 분리 방향.
- **문제 2** (PK: `학번`, `과목코드`, `교수ID` 목표): 1NF에서 다중 교수 행 분해 → 2NF에서 성적 / 학생 / 과목–교수 분리.
  
#### 8. 한 줄 암기
- **1NF**: 원자값.
- **2NF**: 복합키 **부분 종속** 제거.
- **3NF**: **이행 종속** 제거.
- **BCNF**: **결정자 = 후보키**.
- **4NF**: 독립 다중값 **섞지 않기**.

### SELECT

#### 1. 기본 SELECT
- **전체 조회**
  - `SELECT * FROM usertbl;`
  - `SELECT * FROM buytbl;`
- **특정 컬럼만 조회**
  - `SELECT userid, birthYear FROM usertbl;`
- **별칭(alias)**
  - `SELECT userid AS '아이디', birthYear AS '생년월일' FROM usertbl;`
- **문자열 결합(concat)**
  - `SELECT concat(mobile1,'-',mobile2) AS '연락처' FROM usertbl;`
    
#### 2. WHERE 조건절 - 비교 연산자
- **동등 비교**
  - `WHERE name='김경호'`
  - `WHERE userId='LSG'`
- **대소 비교**
  - `WHERE birthYear >= 1970`
  - `WHERE height <= 170`
    
#### 3. WHERE 조건절 - 논리/범위/집합/패턴
- **AND / OR**
  - `WHERE birthYear >= 1970 AND height >= 180`
  - `WHERE birthYear >= 1970 OR height >= 180`
- **BETWEEN**
  - `WHERE height BETWEEN 170 AND 180`
  - (동일 의미) `height >= 170 AND height <= 180`
- **IN**
  - `WHERE addr IN ('서울','경기')`
- **LIKE(패턴)**
  - `LIKE '김%'` : 시작
  - `LIKE '%수'` : 끝
  - `LIKE '%경%'` : 포함
  - `LIKE '김__'` : “김” + 임의 문자(언더바 개수만큼)  
  - (주의) `addr LIKE ('서울','경기')`는 리스트처럼 쓰지 말고 보통 `IN`을 사용
    
#### 4. buytbl 실습: 조건 필터 패턴
- **amount 5 이상**
  - `WHERE amount >= 5`
- **price 50~500**
  - `WHERE price BETWEEN 50 AND 500`
  - (동일 의미) `price >= 50 AND price <= 500`
- **amount 10 이상 OR price 조건**
  - `WHERE amount >= 10 OR price <= 100`
- **문자 패턴**
  - `WHERE userid LIKE 'K%'`
  - `WHERE userid LIKE '%W'`
- **groupName 비어있지 않기 / NULL 체크**
  - 빈 문자열 제외: `groupName != ''`, `groupName <> ''`
  - NULL 제외: `groupName IS NOT NULL`
    
#### 5. 서브쿼리(Subquery) + ALL/ANY
- **단일값 서브쿼리**
  - 예) `height > (SELECT height ... name='김경호')`
- **ALL / ANY**
  - `ALL` : “모든 결과를 만족”(AND 느낌)
    - 예) `height > ALL(SELECT height ... addr IN('경남'))`
  - `ANY` : “하나라도 만족”(OR 느낌)
    - 예) `height > ANY(SELECT height ... addr IN('경남'))`
- **서브쿼리 null 비교**
  - `= ''` 대신 `IS NULL` 사용
    
#### 6. ORDER BY (정렬)
- **ASC(오름차순) / DESC(내림차순)**
  - `ORDER BY mDate asc`
  - `ORDER BY mDate desc`
- **조건 + 정렬**
  - `WHERE birthYear >= 1970 ORDER BY mDate desc`
- **다중 정렬**
  - `ORDER BY height desc, mDate asc`
- **추가 예시(0408_select.sql)**
  - `ORDER BY amount asc, prodName desc`
    
#### 7. DISTINCT (중복 제거)
- **단일 컬럼 중복 제거**
  - `SELECT DISTINCT addr FROM usertbl;`
  - `SELECT DISTINCT prodName FROM buytbl ORDER BY prodName asc;`
  - `SELECT DISTINCT userid FROM buytbl;`
    
#### 8. LIMIT (개수 제한)
- **처음 N개**
  - `LIMIT 3`
- **앞에서 N개 건너뛰고 M개**
  - `LIMIT 2,5`
    
#### 9. 테이블 복사(CREATE TABLE AS / LIKE / INSERT SELECT)
- **구조 + 값 복사**
  - `CREATE TABLE tbl_buy_copy (SELECT * FROM buytbl);`
- **선택 컬럼만 복사**
  - `CREATE TABLE tbl_buy_copy2 (SELECT userid, prodname, amount FROM buytbl);`
- **구조만 복사(like)**
  - `CREATE TABLE tbl_buy_copy3 LIKE buytbl;`
- **데이터만 삽입(insert into ... select)**
  - `INSERT INTO tbl_buy_copy3 SELECT * FROM buytbl WHERE amount >= 3;`
- **조건 데이터 복사**
  - `CREATE TABLE CUsertbl (SELECT * FROM usertbl WHERE addr IN('서울','경기'));`
    
#### 10. GROUP BY (집계)
- **집계함수(SUM / AVG / MIN / MAX) + group by**
  - `SELECT userid, SUM(amount) AS '구매총량' FROM buytbl GROUP BY userid;`
  - `SELECT userid, SUM(amount*price) AS '구매총액' FROM buytbl GROUP BY userid;`
  - `SELECT userid, TRUNCATE(AVG(amount*price),2) AS '구매평균액' FROM buytbl GROUP BY userid;`
  - `SELECT MAX(height) FROM usertbl;`
  - `SELECT MIN(height) FROM usertbl;`
- **최대/최소값 서브쿼리로 행 조회**
  - `height = (SELECT MAX(height) FROM usertbl)`
  - `height = (SELECT MIN(height) FROM usertbl)`
  - (둘 다) `OR`로 같이 출력
- **GROUP BY 문제 예시(0408_select2.sql)**
  - buytbl에서 `userid`별 구매량 합:
    - `SELECT userid, groupname, SUM(amount) AS '합' FROM buytbl GROUP BY userid, groupname;`
  - buytbl에서 키 평균:
    - `TRUNCATE(AVG(height),2)`
  - groupname 개수:
    - `COUNT(groupname) AS 'groupname개수'`
  - world DB 예시
    - `city`: countrycode별 인구 합
    - `country`: continent별 기대수명 평균
      
#### 11. 1문장 암기(집계)
- **SELECT는 “FROM에서 가져오고 → WHERE로 거르고 → GROUP BY로 묶고 → ORDER BY로 정렬하고 → LIMIT로 자른다” 흐름으로 외워라.**
- **집계는 `GROUP BY`가 기준**
- **집계는 `SUM/AVG/MIN/MAX/COUNT`로 요약**
- **소수점은 `TRUNCATE(x, n)`로 자르기**
