### INSERT / CONSTRAINT-제약조건 요점정리

#### 1. INSERT - 여러 행 한번에 삽입

- **여러 값 한 번에 넣기**
  - `INSERT INTO tbl_buy_2 VALUES (1,'aaa','운동화',1), (2,'bbb','운동화',1), ...;`

#### 2. INSERT IGNORE (PK 중복 시 무시)

- **중복 키 에러를 내지 않고 “해당 행을 건너뜀”**
  - `INSERT IGNORE INTO tbl_buy_2 VALUES(1,'aaa','운동화',2);`

#### 3. AUTO_INCREMENT

- **PK를 자동 증가로 넣고 싶을 때**
  - `VALUES(NULL, ...)` 형태로 입력
- **다음 AUTO_INCREMENT 값 확인**
  - `SELECT auto_increment FROM information_schema.tables WHERE table_schema='shopdb' AND table_name='tbl_buy_2';`
- **마지막으로 성공한 auto_increment 값 확인**
  - `SELECT last_insert_id();`
- **AUTO_INCREMENT 초기화(재설정)**
  - `ALTER TABLE tbl_buy_2 AUTO_INCREMENT = 0;`  
    (보통 테이블 비운 뒤 적용)

#### 4. ON DUPLICATE KEY UPDATE

- **PK/UK 중복이면 INSERT 대신 UPDATE 수행**
  - `INSERT INTO tbl_buy_2 VALUES(1,'aaa','운동화',2) ON DUPLICATE KEY UPDATE amount=amount+1;`

> 정리:  
> - “중복이면 그냥 무시” = `INSERT IGNORE`  
> - “중복이면 갱신” = `ON DUPLICATE KEY UPDATE`

#### 5. PRIMARY KEY (PK)

- **테이블 생성 시 PK 지정(컬럼 inline - 컬럼 옆에 바로 쓰기)**
  - `id VARCHAR(45) PRIMARY KEY`
- **테이블 생성 시 PK 지정(테이블 레벨 - 컬럼들 다 쓰고, 마지막에 지정)**
  - `PRIMARY KEY(id)`
- **복합키 PK**
  - `PRIMARY KEY(id, name)`
- **기존 테이블에 PK 추가**
  - `ALTER TABLE tbl_d ADD CONSTRAINT PK_tbl_d PRIMARY KEY(id);`
- **PK 제거**
  - `ALTER TABLE tbl_d DROP PRIMARY KEY;`

- **실습 예시**
  - `CREATE TABLE C_buytbl (SELECT * FROM buytbl);`
  - `ALTER TABLE C_buytbl ADD CONSTRAINT PK_C_buytbl PRIMARY KEY(num);`

#### 6. FOREIGN KEY (FK)

- **테이블 생성 시 FK**
  - `CONSTRAINT FK_xxx FOREIGN KEY(id) REFERENCES tbl_a(id)`

- **기존 테이블에 FK 추가**
  - `ALTER TABLE tbl_test3 ADD CONSTRAINT ... FOREIGN KEY(id) REFERENCES tbl_c(id) ...;`

- **복합키에 대한 FK**
  - `FOREIGN KEY(id,name) REFERENCES tbl_e(id,name)`

- **FK 옵션(핵심 의미)**
  - `CASCADE`     : PK 변경/삭제 시 FK도 같이 변경/삭제
  - `RESTRICT`    : 참조 중이면 PK 변경/삭제 차단
  - `NO ACTION`   : 상황에 따라 제한(엔진 처리) / 실무에선 RESTRICT와 비슷하게 이해하는 경우 많음
  - `SET NULL`    : PK 변경/삭제 시 FK를 NULL로
  - `SET DEFAULT` : PK 변경/삭제 시 FK를 DEFAULT 값으로

- **FK 제약 확인(메타데이터)**
  - `information_schema.referential_constraints` 조회

- **FK 삭제**
  - `ALTER TABLE tbl_test1 DROP FOREIGN KEY FK_tbl_test1_tbl_a;`
  - FK 삭제 후 남아있는 인덱스는 별도 제거 필요할 수 있음:
    - `ALTER TABLE tbl_test1 DROP INDEX FK_tbl_test1_tbl_a;`
  - FK 이름 모를 때:
    - `SHOW CREATE TABLE tbl_test2;`

- **참조 관계에서 테이블 삭제**
  - PK(참조당하는) 테이블은 FK(참조하는) 테이블을 먼저 삭제해야 함.
  - 강제 삭제(주의):
    - `SET foreign_key_checks = 0;` → DROP → `SET foreign_key_checks = 1;`

- **실습 예시(Restrict/Cascade)**
  - `copy_buytbl` 만들고
    - `num`을 PK
    - `userid`를 FK로 설정 (`ON UPDATE CASCADE`, `ON DELETE RESTRICT`)

#### 7. UNIQUE 제약조건

- **중복 불가, NULL은 허용(일반적으로)**
  - 컬럼에 바로 `UNIQUE`
  - 또는 `CONSTRAINT uk_email UNIQUE(email)`
- **기존 테이블에 UNIQUE 추가**
  - `ALTER TABLE tbl_test07 ADD CONSTRAINT Uk_tbl_test07_email UNIQUE(email);`
- **UNIQUE 확인**
  - `SHOW INDEX FROM ...`
  - `SHOW CREATE TABLE ...`

#### 8. CHECK 제약조건

- **값 범위/집합 제한**
  - `age INT CHECK(age>=10 AND age<=50)`
  - `addr`는 특정 값만: `CHECK(addr IN('서울','대구','인천'))`
- **CHECK 확인**
  - `INFORMATION_SCHEMA.CHECK_CONSTRAINTS`
- **CHECK 삭제**
  - `ALTER TABLE tbl_test08 DROP CHECK CK_ADDR;`

#### 9. DEFAULT

- **기본값 지정**
  - `name VARCHAR(20) DEFAULT '이름없음'`
  - `age INT ... DEFAULT 20`
- **DEFAULT 변경/삭제**
  - `ALTER TABLE ... ALTER COLUMN name SET DEFAULT '홍길동';`
  - `ALTER TABLE ... ALTER COLUMN age DROP DEFAULT;`

#### 10. 1문장 암기
- **PK는 유일·NOT NULL, FK는 참조(부모가 먼저)이며 ON UPDATE/DELETE 옵션(CASCADE/RESTRICT/SET NULL)로 연쇄 동작을 제어한다.**
- **데이터 무결성은 “PK로 유일성, FK로 참조무결성, UNIQUE/CHECK/DEFAULT/NOT NULL로 입력 규칙”을 강제해서 지킨다.**
