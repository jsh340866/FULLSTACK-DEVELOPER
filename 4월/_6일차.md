### INSERT / 제약조건 요점정리

#### 1. INSERT - 여러 행 한번에 삽입

- **여러 값 한 번에 넣기**
  - `INSERT INTO tbl_buy_2 VALUES (1,'aaa','운동화',1), (2,'bbb','운동화',1), ...;`

#### 2. INSERT IGNORE (PK 중복 시 무시)

- **중복 키 에러를 내지 않고 “해당 행을 건너뜀”**
  - `INSERT IGNORE INTO tbl_buy_2 VALUES(1,'aaa','운동화',2);`

#### 3. AUTO_INCREMENT

- **PK를 자동 증가로 넣고 싶을 때**
  - `VALUES(NULL, ...)` 형태로 입력
- **다음 AUTO_INCREMENT 값 확인**
  - `SELECT auto_increment FROM information_schema.tables WHERE table_schema='shopdb' AND table_name='tbl_buy_2';`
- **마지막으로 성공한 auto_increment 값 확인**
  - `SELECT last_insert_id();`
- **AUTO_INCREMENT 초기화(재설정)**
  - `ALTER TABLE tbl_buy_2 AUTO_INCREMENT = 0;`

#### 4. ON DUPLICATE KEY UPDATE

- **PK/UK 중복이면 INSERT 대신 UPDATE 수행**
  - `INSERT INTO tbl_buy_2 VALUES(1,'aaa','운동화',2) ON DUPLICATE KEY UPDATE amount=amount+1;`

> 정리:  
> - “중복이면 무시” = `INSERT IGNORE`  
> - “중복이면 갱신” = `ON DUPLICATE KEY UPDATE`

#### 5. PRIMARY KEY (PK)

- **컬럼 inline**
  - `id VARCHAR(45) PRIMARY KEY`
- **테이블 레벨**
  - `PRIMARY KEY(id)`
- **복합키 PK**
  - `PRIMARY KEY(id, name)`
- **기존 테이블에 PK 추가/제거**
  - `ALTER TABLE ... ADD CONSTRAINT ... PRIMARY KEY(...)`
  - `ALTER TABLE ... DROP PRIMARY KEY`

#### 6. FOREIGN KEY (FK)

- **테이블 생성 시 FK**
  - `CONSTRAINT FK_xxx FOREIGN KEY(id) REFERENCES tbl_a(id)`
- **기존 테이블에 FK 추가**
  - `ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY(...) REFERENCES ...`
- **복합키 FK**
  - `FOREIGN KEY(id,name) REFERENCES tbl_e(id,name)`

- **FK 옵션 의미**
  - `CASCADE` : 부모(PK) 변경/삭제 시 자식(FK)도 연쇄 반영
  - `RESTRICT` : 참조 중이면 변경/삭제 차단
  - `SET NULL` : FK를 NULL로
  - (기타) `NO ACTION`, `SET DEFAULT`

- **FK 확인/삭제**
  - 확인: `information_schema.referential_constraints`, `SHOW CREATE TABLE ...`
  - 삭제: `ALTER TABLE ... DROP FOREIGN KEY ...;`
  - 필요 시 인덱스도: `ALTER TABLE ... DROP INDEX ...;`

- **참조 관계에서 DROP 주의**
  - FK 테이블(자식) 삭제 후 PK 테이블(부모) 삭제
  - 강제: `SET foreign_key_checks = 0;` (주의)

#### 7. UNIQUE / CHECK / DEFAULT (알아두기)

- **UNIQUE**: 중복 금지(보통 NULL 허용)
- **CHECK**: 값 범위/집합 제한 (예: `age 10~50`, `addr IN(...)`)
- **DEFAULT**: 값 미입력 시 기본값

#### 8. 1문장 암기

- **PK는 유일·NOT NULL, FK는 참조(부모가 먼저)이며 ON UPDATE/DELETE 옵션(CASCADE/RESTRICT/SET NULL)로 연쇄 동작을 제어한다.**


### 알아두기(간단 요약)

#### 9. 내장함수(문자/날짜) - 자주 쓰는 것만

- **문자 합치기**
  - `concat(...)`, `concat_ws(구분자, ...)`
- **문자열 자르기**
  - `substring(문자열, 시작)` / `substring(문자열, 시작, 길이)`
  - `substring_index(문자열, 구분자, n)`
- **기본 정리**
  - `length()`, `trim()`, `replace()`
- **날짜/시간**
  - `year()`, `month()`, `day()`
  - `now()`, `curdate()`, `curtime()`
- **간단 포맷 흉내**
  - `replace(curdate(),'-','#')`, `replace(curtime(),':','|')` + `concat(...)`

#### 10. 변수 + PREPARE(동적 SQL)

- **세션 변수**
  - `set @var1 = 5;`
- **바인딩해서 실행**
  - `prepare ... from '... limit ?'`
  - `execute ... using @rowcnt;`

#### 11. 형변환(CAST)

- **명시적 형변환**
  - `cast('2024$01$01 12:12:12' as datetime)`
  - `cast('2024@01@01' as date)`
- **출력 문자열 만들 때**
  - `concat(cast(price as char(10)), ' X ', cast(amount as char(10)))`
- **자동 형변환 주의**
  - `'100' + 200` 같은 케이스는 동작은 하지만 의도와 다를 수 있음

#### 12. 업로드/다운로드(파일 ↔ DB) 개념

- **업로드(파일 → DB)**
  - `load_file('c:\\sql\\test2.png')`
  - 보관: `BLOB/LONGBLOB`
- **다운로드(DB → 파일)**
  - `select filedata ... into dumpfile 'c:\\sql\\down1.exe';`
- **주의**
  - `secure_file_priv` 경로/권한 영향
