### JOIN (`0410_JOIN(중요).sql`)

#### 1. 핵심 개념

- **JOIN**: 2개 이상 테이블을 키로 연결해서 한 결과셋으로 조회

#### 2. 꼭 외울 패턴

- **INNER JOIN**
  - `FROM A INNER JOIN B ON A.key = B.key`
- **별칭(Alias)**
  - `FROM usertbl U INNER JOIN buytbl B ON U.userid = B.userid`
- **JOIN + WHERE**
  - 조인 후 조건 필터링: `... WHERE amount >= 3`

#### 3. OUTER JOIN 요점

- **LEFT OUTER JOIN**: 왼쪽 테이블 전체 + 오른쪽 매칭
- **RIGHT OUTER JOIN**: 오른쪽 테이블 전체 + 왼쪽 매칭
- **FULL OUTER JOIN**
  - MySQL은 직접 지원 X → `LEFT JOIN ... UNION RIGHT JOIN ...`로 구현

### Transaction (`0410_TRANSACTION(중요).sql`)

#### 1. 핵심 개념

- 여러 작업을 **하나의 단위로 묶어** 성공 시 `COMMIT`, 실패 시 `ROLLBACK`으로 되돌림(원자성/일관성)

#### 2. 기본 문법 흐름

- `START TRANSACTION;`
- DML 실행(INSERT/UPDATE/DELETE)
- `COMMIT;` 또는 `ROLLBACK;`

#### 3. SAVEPOINT(부분 롤백)

- `SAVEPOINT s1; ... SAVEPOINT s2;`
- `ROLLBACK TO s2;` 처럼 **중간 지점까지** 되돌릴 수 있음

#### 4. 예외 시 롤백(핸들러) 포인트

- 프로시저/트랜잭션 내부에서
  - `DECLARE ... HANDLER FOR SQLEXCEPTION ... ROLLBACK;`
  - 형태로 **에러 발생 시 자동 롤백** 처리

### Stored Procedure (`0410_PROCEDURE(중요).sql`)

#### 1. 핵심 개념

- **Stored Procedure(프로시저)**: SQL들을 논리적으로 묶어서 DB에 저장하고 `CALL`로 실행하는 저장 프로그램

#### 2. 기본 작성/호출

- `DELIMITER $$`
- `CREATE PROCEDURE ... BEGIN ... END $$`
- `DELIMITER ;`
- `CALL proc_name(...);`

#### 3. 자주 나오는 포인트(요점)

- **IN 파라미터로 조건 받기**
  - amount 기준 조회, 범위 조회(예: `BETWEEN arg1 AND arg2`)
- **흐름제어**
  - `IF ... THEN ... ELSE ... END IF`
  - `CASE WHEN ... THEN ... END`
  - `WHILE ... DO ... END WHILE`
- **기준값(평균 등) 비교 라벨링**
  - 평균(`AVG(amount*price)`) 기준으로 `IF`를 써서 ‘평균이상/이하’ 같은 결과 컬럼 만들기
- **날짜함수 응용**
  - `DATEDIFF(CURDATE(), mDate)`로 근속/가입 경과일 계산
  - `DATE(CONCAT(birthyear,'-01-01'))`로 생년 기반 날짜 생성 후 나이 계산

### View (`0410_VIEW(중요).sql`)

#### 1. 핵심 개념

- **View(뷰)**: 실제 데이터를 저장하지 않는 **가상 테이블(쿼리 저장)**
- **목적**
  - 컬럼/행 접근 제한(보안/편의)
  - 복잡한 조인/가공 결과를 재사용

#### 2. 생성/확인/조회

- 생성: `CREATE OR REPLACE VIEW view_name AS SELECT ...`
- 조회: `SELECT * FROM view_name;`
- 정의 확인: `SHOW CREATE VIEW view_name;`
- 목록 확인: `SELECT * FROM information_schema.views WHERE table_schema='shopdb';`

#### 3. 예제에서의 포인트

- `CONCAT(mobile1,'-',mobile2)` 같은 **가공 컬럼**을 뷰에 포함
- `WHERE addr IN ('서울','경기')`처럼 **필터링된 뷰**
- `INNER JOIN` 결과를 뷰로 만들어 **조인 결과 재사용**

### (추가) 알아두는 정도 정리

#### 예외처리 (`0410_EXCEPTION.sql`)

- **핵심 목적**: 에러로 실행이 끊기지 않게 흐름 제어/로그/롤백
- **패턴**
  - 특정 코드 처리: `DECLARE CONTINUE HANDLER FOR 1146 ...`
  - 전체 예외 처리: `... FOR SQLEXCEPTION ...`
  - 에러 확인: handler 내부 `SHOW ERRORS;`
- **에러 로그 남기기(실무 포인트)**
  - `GET DIAGNOSTICS CONDITION 1 ... MYSQL_ERRNO, MESSAGE_TEXT`
  - 로그 테이블에 `NOW()`와 함께 저장
- **트랜잭션과 같이 쓰면**
  - `EXIT HANDLER`에서 `ROLLBACK` → 실패 시 전체 취소

#### JSON (`0410_JSON_Object.sql`)

- 생성: `JSON_OBJECT(k,v, k,v...)`
- 유효성: `JSON_VALID(json)`
- 조회/검색: `JSON_EXTRACT(json, '$...')`, `JSON_SEARCH(json,'one/all','값')`
- 수정: `JSON_INSERT / JSON_REPLACE / JSON_REMOVE`
- JSON 컬럼도 동일 함수로 조회/가공 가능

#### PIVOT(조건부 집계) (`0410_PIVOT.sql`)

- MySQL은 전용 PIVOT 대신 **조건부 집계**로 “행→열” 요약표 생성
- 패턴: `SUM(IF(조건, 값, 0)) AS '열이름'`
- `GROUP BY` 기준(예: `userid`)
- `WITH ROLLUP`으로 합계 행 생성
- 결과를 `VIEW`로 저장해서 리포트처럼 재사용 가능

#### TRIGGER (`0410_TRIGGER.sql`)

- 이벤트 발생 시 자동 실행(대표: **감사/이력**)
- `AFTER UPDATE/DELETE ... FOR EACH ROW`
- 변경 전 값: `OLD.컬럼`
- 백업 테이블에 `type('수정'/'삭제')`, `NOW()` 같은 메타 정보와 함께 저장
- 확인: `SHOW TRIGGERS;`, `SHOW CREATE TRIGGER ...;`
